UI Logic and Feasibility Mapping

This document translates the DataReliQ_v17 1.html mockup into an implementable product logic specification, using the structure and style of the existing v12 logic companion as reference. The v17 mockup introduces a dedicated setup experience, sidebar navigation, embedded stale-analysis inside Freshness, a Reports workspace, and an AI insights side panel.

Overview

The v17 mockup is a single-page application with two top-level states: a first-run setup experience (#setup-page) and the operational dashboard shell (#dashboard-app). Dashboard navigation is sidebar-driven and each primary area swaps a section panel via switchSection(event, name).

Top-Level Product States

Setup is shown before monitoring is complete. After setup confirmation or skip, the dashboard shell becomes the active experience.

Shared Controls

Date range, compute, export, notifications, and live query scanning are global controls that affect or support multiple modules.

Configuration Model

Settings and module configuration appear both during setup and later in the Settings section, indicating reuse of one canonical configuration layer.

Major v17 Change

Stale-table analysis is no longer a separate main tab. It now lives inside the Freshness area as a drill-down trend and filter experience.
