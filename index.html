2. Grant Data Access (SQL)
DataReliQ requires USAGE permissions on the databases and schemas you wish to monitor for freshness and SLA tracking.
GRANT USAGE ON DATABASE <YOUR_DB> TO APPLICATION "DATARELIQ_APP";
GRANT USAGE ON SCHEMA <YOUR_DB>.<YOUR_SCHEMA> TO APPLICATION "DATARELIQ_APP";

Data Access & Privacy
Read-only · 5 ACCOUNT_USAGE views · No business data access ▼
⚠️ ACCOUNT_USAGE latency: Snowflake refreshes these views with up to 3 hours of lag. DataReliQ reflects activity from up to 3 hours ago. For near-real-time freshness checks, DataReliQ also queries INFORMATION_SCHEMA.TABLES (seconds latency, 7-day history).

Views DataReliQ reads from your account

Snowflake ViewSchemaUsed ForSensitivity
QUERY_HISTORYSNOWFLAKE.ACCOUNT_USAGEFailure reasons, top expensive queries, execution timeline, queue latency, cost-per-queryQuery metadata + SQL text
WAREHOUSE_METERING_HISTORYSNOWFLAKE.ACCOUNT_USAGEMonthly cost, 12-month trend, warehouse share, idle credit waste, burn rateCredit data only
TABLE_STORAGE_METRICSSNOWFLAKE.ACCOUNT_USAGEFreshness score, stale tables, most stale table KPITable timestamps only
ACCESS_HISTORYSNOWFLAKE.ACCOUNT_USAGEDownstream impact — which roles and processes depend on each tableAccess patterns by role
USAGE_IN_CURRENCY_DAILYSNOWFLAKE.ACCOUNT_USAGEUSD cost conversion for all cost metricsCredit/cost figures only
ℹ️
DataReliQ never reads your actual business data. All views above are metadata views in Snowflake's ACCOUNT_USAGE schema — they contain information about your queries and tables, not the data inside them. DataReliQ has no SELECT access to any database, schema, or table you own.

How to revoke access

Run as ACCOUNTADMIN in a Snowflake worksheet to fully remove DataReliQ:

-- Revoke DataReliQ access completely
USE ROLE ACCOUNTADMIN;
DROP USER IF EXISTS DATARELI_SVC;
DROP ROLE IF EXISTS DATARELI_MONITOR_ROLE;
-- IMPORTED PRIVILEGES revoked automatically when role is dropped
